# 八字命理知识管理技能 (Documentation Management for Bazi)

## 描述
本技能是 Agent 的"内务总管"与"左脑（写入中心）"，负责维护 `.记忆/`（命理知识库）和 `.项目进展/`（项目演进记录）目录的整洁与有序。
它专注于**写入、结晶与自动化垃圾回收（清理）**，通过脚本强制将临时命理知识和分析记录进行压缩、晋升和归档，消除"知识漂移"与上下文膨胀。

## 适用场景
- **命理知识结晶**: 发现具有复用价值的命理范式（如某种格局的断事模式）、限制（如某种特殊组合的断法）、或通用命理经验后，需要将其结晶到 `记忆` 中。
- **命案分析记录**: 完成一次重要的用户命盘分析后，需要记录分析要点、喜用神校验结果、或特殊的命理现象。
- **项目进展记录**: 排盘脚本更新、新命理体系接入（如紫微斗数）、知识库重构等里程碑完成后。
- **自动化维护（压缩/淘汰）**: 更新了命理知识日志后，调用底层脚本自动评估是否需要归档老旧数据或晋升核心记录。
- **用户指令**: 当用户指令包含"更新文档"、"记录命理知识"、"记录分析"、"收尾"时。

> [!IMPORTANT]
> 此技能**仅用于写入与组织**。如果你需要查找先前的命理知识或分析记录，请调用 `memory-retrieval` 技能。

## 核心职责与触发机制
核心原则：按需求更新，不是每次都要更新所有的文件，只有符合更新需求再进行更新。

### A. 命理知识结晶 (Bazi Knowledge Crystallization) -> `.记忆/`
- **触发**: 
  - 发现重要的命理格局断事模式（如杀印相生、食神生财的特殊用法）
  - 踩过坑并解决（如某个排盘边界情况的处理）
  - Web 搜索获取到 SOTA 命理方案或古籍参考
- **动作**: 将命理经验标准化后写入 `.记忆/当前_记忆.md`。必须符合 `Reference/记忆管理规则.md` 所定义的规则和结构。
- **后置动作**: 写入后必须调用自动化脚本，确保新命理知识被索引，超限时触发归档。

### B. 项目进展追踪 (Progress Tracking) -> `.项目进展/`
- **触发**: 
  - 排盘脚本的关键节点（Milestone）：如新增真太阳时校正、修复节气边界 Bug
  - 新命理体系接入完成：如紫微斗数排盘集成
  - 知识库重大重构或架构变更
  - 微调保持静默（如简单的文案修改无需记录）
- **动作**: 写入 `.项目进展/当前_项目进展.md`。必须符合 `Reference/项目进展管理规则.md` 所定义的规则和结构。
- **后置动作**: 写入后调用自动化脚本，完成晋升（Promotion）、淘汰（Eviction）与压缩。

### C. 全局地图 (`MAP.md`)
- **触发**: 顶级目录结构变更或核心模块入口改变（如新增排盘引擎、新增命理体系）。
- **动作**: 更新树状图，确保 `[功能入口]` 标记指向正确的文件。必须符合 `Reference/CreatMap.md` 所定义的规则和结构。

### D. 局部地图 (`INDEX.md`)
- **触发**: 任何子目录下文件数量 > 3，且发生了文件增删改。
- **动作**: 必须符合 `Reference/CreatMap.md` 所定义的规则和结构。
- **规范**:
  - 必须包含 `File Manifest` 表格。
  - **精准描述**: 严禁使用"辅助工具"等模糊词，必须写明"负责八字四柱排盘计算"或"负责真太阳时经度校正"。
  - **状态标记**: 标注 `Stable`（稳定），`WIP`（开发中），`Deprecated`（弃用）。

### E. 代码头部注释 (`header_comment.md`)
- **触发**: 当新创建一个排盘脚本或命理计算模块时，或者当代码文件头部缺少头部注释时或者格式不对的时候。
- **动作**: 检查代码头部注释是否完整、准确，是否符合 `Reference/HeaderComment.md` 定义的规则和结构。
- **要求**: 信息密度高、逻辑清晰、方便 AI Agent 快速扫读。

### F. 自动化引擎调用 (Auto-Pilot Invocation)
无论更新了哪个"当前_*.md"文件，务必在此后调用对应的维护脚本：
- **统一引擎**: 建议使用 `memory_manager.py`（配置路径指向被修改的 `_当前_` 文件），脚本位于 `skills/documentation-management/scripts` 目录下。
- 脚本将利用 AI 智能化总结并自动生成高密度、双重友好的 `_index.md`。

## 行为准则 (Guiding Principles)
1. **无噪声 (No Noise)**: 只结晶有复用价值的命理知识和架构信息。常规的文案拼写修复不需要记录。
2. **原子化更新 (Atomic Updates)**: 命盘分析或排盘变更落地后，必须在此次对话结束前完成相应文档的更新，不得拖延。
3. **遵循规范 (Strict Schemas)**: 无论是写项目进展还是记命理知识，都必须依据 Reference 里的标准规范来写，这是保证上下文可被智能检索的基础。
4. **纯净输出 (Plain Text Only)**: 严禁使用 Emoji 或任何装饰性特殊 Unicode 字符。仅允许使用标准中英文字符及基础标点，以确保信息在不同终端和检索索引中的兼容性与编码稳定性。
