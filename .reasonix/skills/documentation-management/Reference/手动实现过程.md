# 人工接入技能创建文档

> **说明**：本文档用于指导在人工接入并创建新技能时，需手动执行的维护与初始化命令。

## 1. Documentation-Management (维护与迁移)

### A. 历史数据格式化迁移 (迁移工具)
**功能**：将老版本的加粗格式 (`**VITALITY**`) 转换为最新的 **YAML-Infused SOTA** 格式。

```powershell
# 运行迁移脚本，将 tests 下的原始文件转换为新格式
python tests/migrate_to_new_format.py
```

### B. 智能压缩与索引维护 (主引擎)
**功能**：处理 `.md` 块、提取 YAML、更新索引以及按规则进行归档压缩。

```powershell
# 维护指定的记忆文件
python documentation-management/scripts/memory_manager.py .记忆/当前_记忆.md

# 维护进度文件
python documentation-management/scripts/memory_manager.py .项目进展/当前_项目进展.md
```



### C. 仓库环境初始化 (脚手架)
**功能**：在新项目或新环境下快速建立符合《规则》的目录结构和预留索引表。

```powershell
# 在当前目录下初始化 .记忆 和 .项目进展 空间
python scripts/init_memory_spaces.py .
```

## 2. Memory-Retrieval (检索与存活度)

### A. 语义检索与 Vitality 自动递增 (检索接口)
**功能**：执行查询，并自动触发 YAML 块内 `usage` 和 `last_ref` 的智能更新。

```powershell
# 执行 JSON 参数查询（注意 Windows 下引号的嵌套处理）
python memory-retrieval/scripts/memory_retriever.py '{"query": "LED", "memory_root": "tests", "limit": 2}'
```

## 3. 环境依赖 (关键)

由于使用了结构化的 YAML 读写机制，请确保环境中已安装 `PyYAML`：

```powershell
pip install pyyaml
```
