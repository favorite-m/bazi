# 核心技能：八字命理项目演进记录法则 (Bazi Project Evolution Engine)

你不仅是一个记录者，更是一个**时间维度的命理架构师**。你的核心使命是维护 `.项目进展/` 目录，通过高保真的**"演进切片" (Evolution Slices)**，消除排盘脚本与命理知识库之间的"知识漂移"，确保任何时刻都能回溯命理系统决策的上下文。

## 适用场景 (When to use)
- **结构变更**: 增删排盘脚本文件、重构命理计算模块、新增命理体系目录。
- **接口变更**: 排盘脚本的输入参数/输出格式修改、档案存储格式变更。
- **核心里程碑**: 新排盘引擎上线、新命理体系接入（如紫微斗数）、严重排盘 Bug 修复。
- **用户指令**: 用户明确要求"记录项目进展"或"回顾当前进展"。
- **注意**：微调、文案修复、纯格式化请**保持静默**。

## 1. 演进分类学 (Evolution Taxonomy)

每一次有价值的变更记录均应归属以下类型之一：

| 分类 (TYPE) | 定义 | 示例 |
| --- | --- | --- |
| **MILESTONE** | 关键节点的闭环 / 命理体系级架构变更 | "八字排盘引擎 V2 完成真太阳时校正" |
| **FEATURE** | 新增核心命理功能特性 | "新增紫微斗数排盘能力" |
| **REFACTOR** | 排盘脚本或知识库的重大重构 | "命理知识库由平面结构迁移到层级分类" |
| **HOTFIX** | P0 级别的紧急排盘异常修复 | "修复立春交接时大运起运计算错误" |

*(纯文案修复、格式调整等 `CHORE` 类变动，无需污染本进度表)*

## 2. 存储架构 (Flat-Tiered Storage)

- **根路径**: `.项目进展/`
- **节点定义**:
  - `核心_项目进展.md`: 仅保留**最核心的命理系统架构流变**，即当前系统长效存续的核心节点变动记录。
  - `当前_项目进展.md`: (高频瞬态缓冲) **只记录最近 2-5 天内的冲刺详情**。超过 **500行** 或 **30个条目** 后，自动触发脚本打包抽档。
  - `核心_项目进展_index.md`: (脚本自动维护) 核心里程碑的全局索引。
  - `当前_项目进展_index.md`: (脚本自动维护) 近期进度的全局索引。
  - `存档/历史_项目进展_chunk_*.md`: 冷数据切片日志。
  - `存档/历史_项目进展_index.md`: 冷数据全局索引。

## 3. 标准记录格式 (YAML-Infused Chunking)

在向 `当前_项目进展.md` (或 `核心`) 书写进度时，必须**采用大模型原生理解度最高的 YAML 注入架构**，并加入存活度（Vitality）机制，追加到最前面：

```markdown
##  [ID-项目名称-类别-YYYYMMDD-HHMM] [一句话总结，如：接入真太阳时校正]

```yaml
type: [MILESTONE / FEATURE / REFACTOR / HOTFIX]
vitality: { usage: 0, last_ref: "YYYY-MM-DD" }
status: [已完成 / 推进中 / OBSOLETE (若发现历史冲突)]
tags: [相关脚本, 命理体系, 核心关键字]
```

## 4. 自动化治理 (Auto-Pilot Target)

每次完成进度更新后，**必须**调用唯一的基建底层脚本：
> `python documentation-management/scripts/memory_manager.py ./.项目进展/当前_项目进展.md`

1. **里程碑晋升（Promotion）**: 
   脚本会将 YAML 头中 `type` 为 `MILESTONE` 或 `REFACTOR` 且 `status` 包含 `已完成` 的块，复制晋升至 `核心_项目进展.md`。
2. **瞬态回收（Compaction）**: 
   当前进展超过阈值后，移入当月冷库（如 `2026/04/块...md`）。
3. **墓碑驱逐（Tombstone Eviction 保持环境极度无噪声）**:
   当项目发生了重大的命理逻辑更迭，导致曾经挂在 `核心_项目进展.md` 上的旧架构说明彻底失效，你不需要人工删文字。只要你把针对旧块里的 YAML `status` 修改为 `OBSOLETE`，**它将在执行以上脚本时灰飞烟灭**——从核心区拔除并直接下放至历史冷宫，让核心图谱永远只有最新的真理逻辑！

## 5. 问题处理

当 StrReplace 因文件内含 Unicode（如弯引号 ""）、编码或不可见字符导致匹配失败时，
用本脚本做按行替换或 UTF-8 安全的内容替换，避免在对话里写临时 Python 片段。
> `python scripts/doc_replace_line.py --file .项目进展/当前_项目进展.md --old "某段旧内容" --new "新内容"`
