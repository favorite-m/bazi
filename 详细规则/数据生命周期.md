# 用户数据生命周期管理细则

> 本文件为 `REASONIX.md` 的按需加载子文档，包含完整的数据生命周期操作规范。
> 涉及档案创建、更新、隐私合规时加载此文件。

## 创建与读取

- 首次交互 → 在 `/客户档案/{称呼}/` 下创建标准化档案文件（`0-基础信息.md` + `1-分析记录日志.md` + `0-目录.md`）
- 后续交互 → **必须先读取总目录**判断是否已存在，然后按需加载
- 读取顺序：`0-目录.md` → `0-基础信息.md` → `1-分析记录日志.md`（按需）
- 记忆恢复 → 若需要跨会话检索，调用 `memory-retriever.py` 进行语义搜索

## 更新与持久化

- **每次咨询结束** → 强制更新 `1-分析记录日志.md`，追加本次的分析记录
- **命盘数据** → Skill 返回后立即补全到 `0-基础信息.md` 的"命理数据"部分
- **喜用神修正** → 在 `1-分析记录日志.md` 中记录修正原因，并更新 `0-基础信息.md`
- **档案格式** → 纯文本 Markdown + YAML 元数据头，保证人类与机器均可直接读写

## 隐私合规

- 用户档案数据仅限本地存储与当前会话上下文使用
- 严禁任何形式的跨会话云端持久化
- 严禁将用户数据用于模型微调训练
