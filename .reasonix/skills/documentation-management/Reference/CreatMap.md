在进行八字命理项目极速开发时，为了防止项目复杂度失控并确保你能随时精准定位信息，必须执行以下【双层动态索引】规范：

### 1. 建立真相源：根目录 `MAP.md`
在项目初始化及后续任何结构变动时，必须维护根目录的 `MAP.md`，包含：
- **Project DNA**: 一句话描述项目的核心目标（八字命理分析智能体）与技术栈。
- **Architecture**: 核心模块的树状图，并对每个顶级目录标注【功能入口】。
- **Agent Context**: 告诉接手的 Agent，如果要修改核心命理逻辑，应该从哪几个文件开始看，有对应的 INDEX.md 可以查看。

### 2. 建立局部地图：子目录 `INDEX.md`
在任何包含 3 个文件以上的子目录下，必须自动维护一个 `INDEX.md`：
- **Module Goal**: 该文件夹存在的目的。
- **File Manifest**: 
  | 文件名 | 精准功能 (一句重点) | 状态/重要性 |
  | :--- | :--- | :--- |
  | [File Name] | [该文件具体负责什么命理逻辑] | [Stable/WIP/Entry] |
- **Dependencies**: 该模块依赖了哪些外部模块或库。

### 3. 自动化更新约束 (关键)
- **同步更新**: 每当你【创建、删除、重命名】文件或【大幅修改逻辑】后，必须在同一次对话中更新对应的 `INDEX.md` 或 `MAP.md`。
- **精准描述**: 严禁使用"工具"、"辅助"等模糊字眼，必须写明具体命理业务逻辑（如："负责四柱排盘计算与真太阳时校正"、"负责用户档案的本地持久化存储与查询"）。
